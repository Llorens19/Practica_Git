L'objectiu és mostrar com funciona Git en un entorn de treball col·laboratiu i com s'utilitzen diferents branques per gestionar les versions del codi.

2.- Desenvolupament de la tasca

2.1.- Introducció teòrica

2.1.1.- Què és Git?

Git és un sistema de control de versions que permet gestionar els canvis en el codi i col·laborar entre programadors. Facilita mantindre un historial de modificacions, revertir errors i treballar sense conflictes.

2.1.2.- Què és Git Flow?

Git Flow és una metodologia de treball que organitza el desenvolupament en diferents branques:

  • Master: Conté el codi estable en producció.
  • Develop: Rama de desenvolupament on s'integren noves funcionalitats.
  • Feature: Es creen per afegir noves funcionalitats, que després es fusionen en develop.
  • Release: Es creen per preparar el codi abans de llançar una nova versió estable.
  • Hotfix: Es creen per corregir errors urgents en producció.

2.2.- Explicació detallada del procés

2.2.1.- Usuari 1: Creació del repositori i estructura inicial

L'usuari 1, el més experimentat, crea el repositori del projecte i segueix els passos de la metodologia Git Flow:

  1. Crear un nou repositori remot y el clonem al local: git clone https://github.com/Llorens19/Practica_Git.git
  2. Inicialitzar Git Flow: git flow init
  3. Crear una branca feature: git flow feature start estructura_usuari1
  4. Després de finalitzar els commits en aquesta branca, tancar-la: git flow feature finish estructura_usuari1
  5. Enviar els canvis al remot: git push --set-upstream origin develop

2.2.2.- Usuari 2: Creació de features per al contingut HTML

L'usuari 2 s'encarrega de crear dues features, una per a "Modificar contingut HTML" i una altra per a "Modificar atributs HTML". Els passos són:

  1. Clonar les branques i el contingut remot: git pull origin
  2. Crear una branca feature: git flow feature start seccion_modificar_contingut_html
  3. Finalitzar els commits i tancar la branca: git flow feature finish seccion_modificar_contingut_html
  4. Crear una nova feature per a la secció d'atributs: git flow feature start seccion_atr_html_usuari2

2.2.3.- Usuari 3: Creació de feature per als estils CSS i release

L'usuari 3, encarregat dels estils CSS, crea una feature per afegir la seua part del codi i després llança la release:

  1. Crear una branca feature: git flow feature start seccion_css_usuari3
  2. Finalitzar els commits i tancar la branca: git flow feature finish seccion_css_usuari3
  3. Crear una nova branca release: git flow release start v1.0
  4. Després de comprovar que no hi ha errors, tancar la branca: git flow release finish v1.0

2.2.4.- Usuari 1: Creació d'un hotfix

Després que l'usuari 2 completara el seu treball, l'usuari 1 realitza millores a la secció creada per l'usuari 2 mitjançant un hotfix:

  1. Crear una branca hotfix: git flow hotfix start milloresV_1_0
  2. Finalitzar el hotfix i fusionar els canvis: git flow hotfix finish milloresV_1_0
